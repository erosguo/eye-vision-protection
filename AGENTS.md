# AGENTS.md — eye-vision-protection

## Entrypoint & commands

- App entry: `src/main.js` (Electron main process)
- `npm start` — run in dev mode
- `npm run build` — package Windows NSIS installer via electron-builder, output in `dist/`
- No test/lint/typecheck scripts configured
- `.gitignore` does not exist; `dist/` and `node_modules/` should be ignored

## Architecture

Three-layer design with cross-platform intent:

| Layer | Dir | Depends on | Portability |
|-------|-----|------------|-------------|
| **Core** | `src/core/` | Node.js only | Zero Electron deps, reusable anywhere |
| **Adapter** | `src/adapters/` | Electron APIs | Swap out per-platform |
| **UI** | `src/ui/` | Preload IPC | Replace per-platform |

- **StateMachine** (`state-machine.js`): 4 states — `IDLE` → `WORKING` → `PAUSED`/`ON_BREAK` → `WORKING`. Transitions are strict; an invalid action returns `false` silently. Check `can(action)` before calling `transition(action)`.
- **TimerManager** (`timer-manager.js`): Uses `setInterval` at 1s granularity. Ticks only in WORKING or ON_BREAK states. Calls `onWorkComplete`/`onRestComplete` callbacks when counters hit zero. `_startTick()` always stops previous interval first.
- **SettingsManager** (`settings-manager.js`): Reads/writes `settings.json` in `app.getPath('userData')`. Defaults: workInterval=25, restDuration=5, soundEnabled=true, autoStart=false.

## IPC & preload

- `src/preload.js` uses `contextBridge` to expose `window.api.*` in all renderers
- Break overlay renderers (`ui/break/`) use `api.onBreakUpdate()` + `api.sendBreakReady()` + `api.skipBreak()`/`api.snoozeBreak(minutes)`
- Settings renderer (`ui/settings/`) uses `api.getSettings()` / `api.saveSettings(settings)`
- `break-action` IPC is handled by **ScreenBlocker adapter** (not `main.js`), which forwards to `onUserAction` callback
- State broadcasts (`state-changed`, `break-update`) are sent from main process on a 1s interval

## Key adapter behavior

- **ActivityDetector**: Polls `powerMonitor.getSystemIdleTime()` every 5s. Idle ≥120s → `onIdle`; ≥600s → `onLongIdle`; activity resumes → `onActive`. Fires callbacks only on state changes (debounced internally).
- **ScreenBlocker**: `show()` creates one frameless, always-on-top, fullscreen `BrowserWindow` per display via `screen.getAllDisplays()`. Uses `screen-saver` level for highest z-order. `hide()` closes all.
- **TrayManager**: Tray icon from `src/assets/tray-icon.png` (16×16). Switches to `tray-icon-rest.png` during break. Menu: 显示设置 / 暂停计时 / 退出.
- **AudioPlayer**: Uses `shell.beep()` with two rapid bursts (300ms apart). Disabled via `setEnabled(false)` (tied to settings toggle).

## UI pages

- **Break overlay** (`ui/break/`): Fullscreen dark background, gradient title, 96px countdown timer, progress bar, three action buttons. All text is Chinese.
- **Settings** (`ui/settings/`): 480×520 non-resizable window. Range sliders (workInterval: 5-120 step 5, restDuration: 1-30 step 1), toggle switches, save button with brief success flash.

## Icons

- Generated by `scripts/generate-icons.js` (pure Node.js, zero deps — uses raw PNG byte construction with zlib)
- `src/assets/icon.png` (256×256, app icon), `src/assets/tray-icon.png` (16×16, tray normal), `src/assets/tray-icon-rest.png` (16×16, tray during break, green tint)

## Gotchas

- Electron binary download may fail behind firewalls; set `ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/` for China region
- No framework bundler — UI pages are plain HTML files loaded via `BrowserWindow.loadFile()`
- TimerManager `workRemaining`/`restRemaining` are in seconds (not milliseconds), derived from settings in minutes × 60
- During timer pause (idle 120s+), tick interval is cleared; `resume()` creates a fresh interval
- Long idle (600s+) calls both `onLongIdle` and `onIdle`; main.js responds by calling `abort()` which transitions to IDLE — timer must be manually `start()`ed again
